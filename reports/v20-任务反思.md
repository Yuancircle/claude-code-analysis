## 🛡️ 步骤 3：建立保证机制

### 核心问题
> "我如何保证后续做类似的事情能完全按照要求来？"

### 根本原因分析

**问题 1：阶段跳跃执行**
- 现象：阶段 2 完成后直接跳到阶段 6，阶段 3-5 未执行
- 原因：任务阶段依赖关系未强制执行，前置条件检查缺失
- 教训：阶段 6 的前置条件是"阶段 5 完成后"，但此条件未被验证

**问题 2：进度追踪不透明**
- 现象：任务文件显示"状态：🔄 阶段 2 执行中"，但阶段 6 仍被触发
- 原因：任务状态更新不及时，子代理无法准确判断前置条件
- 教训：任务状态需要实时同步，前置条件需要显式验证

**问题 3：验收标准未量化检查**
- 现象：最终文档大小（≥200KB）、行数（≥8,000）未检查
- 原因：缺少自动化检查机制
- 教训：验收标准需要代码化检查，不能依赖人工判断

### 保证机制

#### 1. 规范保证 → 创建任务阶段依赖规范

**文档：** `life/areas/requirements/task-stage-dependency.md`

**内容：**
- 任务阶段必须有明确的前置条件
- 子代理启动前必须验证前置条件（检查任务文件状态）
- 阶段状态必须实时更新（每完成一个阶段立即更新任务文件）
- 阶段跳过需要显式审批

#### 2. 流程保证 → 修改任务执行流程

**流程改进：**
```
阶段 N 启动前检查清单：
□ 阶段 N-1 状态为"✅ 完成"
□ 阶段 N-1 输出文件已生成
□ 阶段 N-1 验收标准已验证
□ 任务文件已更新阶段状态
```

**任务文件模板更新：**
```markdown
### 阶段 N：[阶段名] [ ]
- [ ] 前置条件验证：阶段 N-1 ✅
- [ ] 子任务 1
- [ ] 子任务 2
- [ ] 验收标准验证
- [ ] 状态更新：✅ 完成（时间戳）
```

#### 3. 检查保证 → 创建阶段依赖检查脚本

**脚本：** `scripts/check-task-stage-dependency.sh`

```bash
#!/bin/bash
# 检查任务阶段依赖
TASK_FILE=$1
CURRENT_STAGE=$2

# 读取任务文件，检查前置阶段状态
PREV_STAGE=$((CURRENT_STAGE - 1))
if grep -q "阶段 $PREV_STAGE.*✅" $TASK_FILE; then
    echo "✅ 前置条件满足：阶段 $PREV_STAGE 已完成"
    exit 0
else
    echo "❌ 前置条件不满足：阶段 $PREV_STAGE 未完成"
    exit 1
fi
```

#### 4. 自动化保证 → 验收标准代码化

**脚本：** `scripts/verify-task-acceptance.sh`

```bash
#!/bin/bash
# 验收标准自动验证
DOC_FILE=$1
MIN_SIZE_KB=200
MIN_LINES=8000

# 检查文件大小
ACTUAL_SIZE_KB=$(du -k $DOC_FILE | cut -f1)
ACTUAL_LINES=$(wc -l < $DOC_FILE)

if [ $ACTUAL_SIZE_KB -ge $MIN_SIZE_KB ] && [ $ACTUAL_LINES -ge $MIN_LINES ]; then
    echo "✅ 验收标准满足：${ACTUAL_SIZE_KB}KB ≥ ${MIN_SIZE_KB}KB, ${ACTUAL_LINES}行 ≥ ${MIN_LINES}行"
    exit 0
else
    echo "❌ 验收标准不满足：${ACTUAL_SIZE_KB}KB < ${MIN_SIZE_KB}KB 或 ${ACTUAL_LINES}行 < ${MIN_LINES}行"
    exit 1
fi
```
